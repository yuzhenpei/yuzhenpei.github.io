Ubuntu

  • pwd：查看当前目录
  • cd ：切换目录 / 开头绝对路径 . 当前目录 ../返回上级目录
  • mkdir ： 创建目录 -p 创建多级目录
  • whoami : 查看当前用户
  • stat ： 查看文件相关属性
  • which ： 查看文件的路径
  • chmod: 修改文件权限 chmod 444 test // 4 2 1 w r x
  • file ： 查看文件类型
  • rmdir：删除目录 （不能删除非空目录） rm -rf 可以删除非空目录
  • ls ： 查看信息 ls -l 查看详细信息 ls -a 查看全部信息 ls -lh h表示内存大小
  • touch : 创建文件，如果存在就更新文件时间
  • cp ： v显示进度 f 强制 r 递归 （目录拷贝需要递归）
    • cp file dir/ 如果dir下有同名文件默认覆盖
    • cp -i file dir/ 提示是否覆盖
    • cp file1 file2 dir/ 将多个文件拷贝到
    • cp -r dir1 dir2 将目录1 拷贝到目录2中
  • rm ：删除文件或目录 （默认强制删除）
    • -i：提示是否删除
    • -r：递归的删除目录下的内容，删除文件夹必须加这个
    • -f：强制删除
  • mv : 移动文件或目录
    • -i : 提示是否覆盖
    • -f ：强制
    • -v：显示进度
    • mv file1 file2 将file1改名为file 2
    • mv file dir/ 将file移到dir下
  • cat ：将文件信息输出给终端
    • -n ：对输出的所有进行编号
    • -b ： 给输出的非空行进行标号
    • -A : 会打出换行符
  • more （不推荐使用）：查看，不能回退查看文件
  • less：将文件内容分页展示给终端
    • ctrl+n 向下滑
    • ctrl +p 向上滑
    • q 退出
  • head ： 查看文件从开始到第n行
    • head -n 文件
    • -c 查看多少个字符
    • -f 动态显示
    • -q ：不显示头信息
    • -v 显示头信息
  • tail ：从文件底部开始查看
    • tail -n 5 file 查看文件最后五行
    • 没有指定n的值的话，默认查看最后十行，其他跟head一样
  • du ：查看目录大小
    • du -sh 文件 查看指定文件或者数据块的大小并加上单位
    • -a 递归显示指定目录个文件和子目录所占用的数据块
    • -s
    • -b 以字节为单位显示磁盘
    • -h 以k，m，g为单位
  • df ： 查看文件磁盘空间占用和可用情况
    • df -h
  • find： 非常实用，查看文件
    • find + 路径+ -name +“文件名”
    • find+路径+ -size +范围 +100k表示大于100k，-100k表示小于100k，k小写，M大写 -size +50k -size -100k 50k到100k之间
    • find + 路径+ -type +类型
      普通文件是用f 而不是_ d目录 l 符号链接 c块设备文件 c字设备文件 s socket文件 p管道
  • grep ： 匹配进行查找
    • grep + 选项+ ‘ 搜索的内容串 ’ + 文件名
      • -i 忽略大小写
      • -v 不包含搜索的内容
      • -n 包含搜索的内容的行号
      • 搜索目录的时候需要加 -r
      • -a 二进制文件
      • -w 完全匹配的那一行
  • 管道 | ：一个命令的输出可以作为另一个命令的输入
  • tar： linux中最常用的备份工具，把一系列文档归档到一个大文件中，也可以将档案文件解开恢复数据
    格式：tar 选项打包文件名 文件
    • -c 生成档案文件 ，创建打包文件
    • -v 列出打包的过程 进度
    • -f 指定档案文件名称 ，f 后面一定跟的是.tar文件，所以必须放选项最后
    • -t 列出档案中包含的文件
    • -x 解开档案文件
      一般组合使用：
    • tar -cvf yzpcode.tar yzpcode // 打包
    • tar -xvf yzpcode.tar // 将打包的文件归档还原
    • tar -tvf yzpcode.tar // 查看里面是否有自己需要的文件
  • gzip ：tar与gzip命令结合使用 实现文件打包和压缩 大文件一般使用gzip
    • 默认是压缩 后面直接跟需要压缩的文件，不保留原文件压缩
    • tar 只负责打包文件不负责压缩，gzip负责压缩tar打包后的文件 ， 其扩展名 一般为 ##.tar.gz
    • 格式： gzip 选项 被压缩的文件
      • -d 解压
      • -r 压缩所有子目录
    • 目录一定要先打包在压缩
    • 组合使用：
      • tar -czvf yzpcode.tar.gz yzpcode // 先打包再进行压缩 可以指定压缩到哪里需要使用绝对路径
      • tar -xzvf yzpcode.tar.gz //对压缩后的文件进行解压 如果需要指定解压到哪一个目录 在后面 加 -C 路径
  • bzip2 ： 跟gzip 相同 后缀名不一样为 .bz2 小文件一般使用 bzip2
    • 结合tar ： 压缩 ： tar -jcvf yzpcode.tar.bz2 yzp
    • 解压：tar -jxvf yzpcode.tar.bz2
    • 万能解压 ： tar -xvf 万能解压（可以解压gzip和bzip2）
  • zip 和 unzip ：
    • zip myzip 文件… myzip就是压缩文件名称 不需要加后缀，后面的文件可以跟多个，打包和压缩到myzip
    • unzip myzip.zip 解压文件 unzip -d 路径 myzip.zip 加压文件到指定路径
  • 访问权限：
    • r ： 读权限
      • 文件：具有读取文件内容的权限
      • 目录： 具有浏览目录的权限
    • w： 写权限
      • 文件：具有新增，修改的权限
      • 目录：具有删除，移动目录内文件的权限
    • x ： 可执行权限
      • 文件：具有文件执行的权限
      • 目录：该用户有进入目录的权限
        只允许文件的所属者或者超级管理员可以修改文件的读写权限
  • chmod （需要熟练）：改变文件权限
    • 字母法：chmod u/g/o/a +/-/= rwx
      • u : 文件的所有者
      • g ： 与user 同组的
      • o ： 其他以外的人
      • +：新增权限
      • -：撤销权限
      • =：设定权限
        例子：chmod o+w a.txt 为文件a的其他用户增加写的权限
        chmod u=rw,g=r,o=r a.txt 设置user可读写，组内可读，外面可读
    • 数字法：chmod 764 a.txt
      • rwx 111 ====》7
      • rw- 110 =====》 6
      • r– 100 =====》 4
      • r-x 101 ====》5
    • 修改目录 和 里面文件的权限 chmod -R 777 test/
  • chown ： 修改目录所有者 sudo chown root a.txt // 将a.txt 的所属者修改为root
    sudo chown : root a.txt //将a.txt 的所属组改为root
  • 重定向：
    • 标准输出是显示到屏幕上，使用 ls 内容 > 文件名 ，将ls 显示的内容 输入到文件中显示，但是会覆盖文件原本的内容，，如果不想覆盖 就使用 >> 追加到文件里面
    • 标准错误输出，ls dasf （使用2>）2>> 文件 将错误输出到文件里面， 一般丢在黑洞文件里面 /dev/null
  • ln (掌握):
    • 软连接不占磁盘空间，原文件删除软链接失效
    • 硬链接只能链接普通文件，不能链接目录
    • ln 源文件 链接文件 //硬链接
    • ln -s 源文件 链接文件 // 软连接 并没有存储文件的数据，只存储文件的路径 readlink 可以读取符号链接的内容（存储目标文件的路径）
