Makefile

主要解决的时间

  • 大量代码的维护
  • 减少重复编译时间
  • Makefile 命名规则 ： makefile 或者 Makefile
  • make安装 ： sudo apt install make

    Makefile语法规则

  • 一条规则，三要素
  • 规则

    目标：依赖文件目录

    命令列表

  • 三要素
    • 目标： 通常是要产生的文件名称，目标可以 是可执行文件或其他obj文件，也可以是一个动作的名称
    • 依赖文件： 用来输入从而产生目标文件，一个目标通常有有几个依赖文件（可以没有）
    • 命令： make执行的动作，一个规则可以含有几个命令（可以没有），每个命令占一行

      Makefile命令格式

    • make [-f file] [options] [targets]
  1. -f : 指定以上名字以外文件作为Makefile输入文件
  2. [options]:
    • -v 显示make版本信息
    • -w：在处理Makefile之前和之后显示工作路径
    • -C dir ： 在读取Makefile文件之前将工作路径改为dir
    • -n ： 只打印要执行的命令但不执行
    • -s ：执行但不显示执行
  3. [targets]:
    • 若使用make命令的时候没有指定目标，默认为第一个目标文件
    • make aa -f 1.mk

      Makefile中的函数

      两个最常用的
  • wildcard： 查询指定目录下的指定文件

    src = ${wildcard *.c}

    //找到当前目录下所有后缀为.c 的文件，赋值给src

  • patsubst： 匹配替换

    obj = ${patsubst %.c , %.o,${src}}

    //将src变量里面所有.c 后缀的文件全部替换为.o文件

    Makefile中的伪函数

    clean用途: 清除编译生成的中间.o文件和最终目标文件
    make clean 如果当前目录下有同名clean文件，则不执行clean对应的命令，解决方案：
    Ø 伪目标声明: .PHONY:clean
    声明目标为伪目标之后，makefile将不会该判断目标是否存在或者该目标是否需要更新

clean命令中的特殊符号：

  • “-”此条命令出错，make也会继续执行后续的命令。如:“-rm main.o”
  • “@”不显示命令本身,只显示结果。如:“@echo clean done”

总结：一条规则，两个函数，三个变量
