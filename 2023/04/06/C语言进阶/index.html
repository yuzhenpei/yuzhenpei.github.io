C进阶

数据类型

  • 声明数据类型是为了更好的管理内存，为数据分配合理的内存空间
  • 定义的数据都得有一种数据类型
  • 创建变量的模具，固定大小内存的别名，即int a = 10；是分配一个4个字节的内存空间，里面存的数据为10(里面用16进制表示0)，然后用别名 a来操作这段内存

typedef

实际就是起一个别名来操作

  • 简化结构体的struct关键字
  • 区分数据类型 typedef char * Pchar;
  • 提高移植性，当需要修改特定的时候 提前用typedef定义后只需要修改定义就行

void

  • 无类型 无法定义数据类型，编译器无法知道为数据分配多大的内存

  • 限定函数返回值，限定函数参数

  • 万能指针 void *p=NULL;(任何指针用sizeof调用都是四个字节)

    不同类型的指针需要强转才能转换，void指针可以不通过强制类型转换

sizeof

  • 本质不是一个函数，而是一种运算符

  • 在统计类型的时候需要加括号，在统计变量的时候可以不加括号

  • sizeof的返回值是一个unsigned int 无符号整型

    一个unsigned int 和int运算，结果会统一为unsigned int

  • 可以求数组所占内存空间的大小

    当一个数组名传入到函数中作为参数，会被退化为一个指针，这个指针指向该数组首元素的地址，其实数组名就是一个地址

变量的修改

  • 变量本质：一段连续内存空间的别名
  • 不是向变量名读写数据，而是向变量名所代表的内存空间进行读写数据
  • 直接修改
  • 间接修改 ： int *p=&a; *p=20;// *p是解引用过程，是访问p内存存放的地址所代表的值

内存四区

  • 运行之前：

    • 预处理：宏定义展开，头文件展开，条件编译，这里并不会检查语法
    • 编译：检查语法，将预处理后的文件编译成汇编文件
    • 汇编：将汇编文件生成目标文件（二进制文件）
    • 链接：将目标文件链接为可执行文件

  • 代码区：存放cpu执行的机器指令。通常代码区是共享的，共享的目的是频繁被执行的程序只需要在内存中有一份代码就可以，代码区通常是只读的

  • 全局初始化数据区或者叫做静态数据区（data段）：在程序中被明确初始化的全局变量，已经初始化的静态变量和常量数据

  • 未初始化数据区（又叫bss区）：存入的是未初始化变量和未初始化静态变量，未初始化数据区在程序执行前被内核初始化为0或者NULL；

  • 运行之后：

    • 栈区：先进后出，编译器自动分配释放，函数的参数，返回值，局部变量，在生命周期结束的时候自动释放内存，栈容量有限，容易栈溢出
    • 堆区：大容器，用于动态分配内存，堆在内存中位于bss和栈之间，一般由程序员分配和释放，程序结束时由操作系统回收

栈区的注意事项

  • 在函数里面定义的局部变量不能作为返回值，因为在函数运行结束的时候栈中已经把局部变量的内存释放
  • 不要返回局部变量的地址值

堆区以及注意的事项

  • 使用 malloc关键字向堆申请内存空间，在使用完后建议free掉
  • memset 为新申请的内存做初始化 memset（内存，0，大小）
  • 主调函数里面的空指针，被调函数中用同级指针分配内存失败需要高级指针来修饰，在主调函数中把指针的地址传入进去

静态变量和全局变量

  • 静态变量：static 关键字修饰
    • 在程序执行前分配内存
    • 生命周期为整个程序运行周期
    • 默认属于内部链接属性，在当前文件夹里使用
    • 还是跟其他变量的作用域规则一样
  • 全局变量：
    • 在C语言中的全局变量在类型前面默认加了一个extern关键字
    • 属于外部链接属性
    • extern int g_b； 告诉编译器 其他文件中有一个g_b 在链接的时候去别的文件中找

const常量

  • 全局的const 常量 ：

    • 不可修改 ：放在常量区，受到常量区的保护，只具有读的操作

  • 局部的const 常量 （伪常量） ：

    • 不可直接修改，但是可以用指针间接修改

      因为是局部变量 ， 是保存在栈中的，伪常量不可初始化数组

字符串常量

  • char * p = “hello world”;

  • 相同的字符串常量，被不同的指针所指向，指针的地址值相等

    • 字符串常量由编译器来决定是否可以修改

宏函数

  • 在预处理阶段就做了宏的替换
  • 使用场景：短小且频繁使用的函数，封装为宏函数
  • 优点：以空间换时间（普通函数的变量需要频繁入栈和出栈）
  • 注意事项：运算的时候需要保证运算的完整性

函数的调用流程

  • 记录函数的返回地址（在调用完子函数后，能继续执行主函数的后面的代码）
  • 进栈，参数从左往右进栈

调用惯例

  • 主调函数和被调函数之间需要有一个约定才能正确的调用函数

  • cdecl C语言默认的调用惯例，参数传递从右到左参数入栈，出栈方是函数调用方，名字修饰用下划线加函数名

函数变量传递分析

  • main函数在栈中开辟的内存，所有子函数都能使用
  • main函数在堆中开辟的空间，所有子函数均可以使用

栈的生长方向

  • 栈底是高地址，栈顶是地地址

内存存储方式

  • 根据大端对齐模式还是小端对齐模式
  • 大端对齐模式：高位字节数据放在内存低地址处
  • 小端对齐模式：高位字节数据放在内存高地址

指针强化

  • 指针是一种数据类型，占用内存空间，用来保存地址（任何类型的指针都只占四个字节的内存空间）
  • 空指针：指向NULL 野指针：指向的内存不是自己申请的
  • 不能向空指针和野指针指向的内存进行操作，如果操作，程序中断

野指针三种情况

  • 指针没有进行初始化，就进行操作
  • malloc后free后的指针 解决办法：指向NULL
  • 指针变量超出了作用域
  • 空指针可以重复释放，野指针不能重复释放

指针的步长

  • 指针变量+1后跳跃的字节数
  • 在解引用时候，取出的字节数
  • 结构体中成员的偏移量 ： 用offsetof（结构体，成员名）宏函数; 需要导入头文件#include<stddef.h>

指针的间接赋值

  • 间接赋值的三大条件：
    • 两个变量（普通变量或指针变量，形参实参）
    • 建立关系
    • 通过*操作指针指向的内存

指针做函数参数

  • 输入特性：主调函数分配内存，被调函数可以使用
  • 输出特性：被调函数分配内存

字符串指针强化

  • 注意事项：

    • 字符串结束标志 \0

    • char str[] ={‘a’,’c’,’\0’} 这种需要在后面手动加上\0结尾

    • char str[10] = {‘s’,’b’} 这种会自动给你加上\0结尾

    • char str = “hello world” 也是自动加上\0 strlen() 不统计\0 sizeof() 统计\0

字符串拷贝

  • 第一种：根据下标进行拷贝

  • 第二种：根据指针进行拷贝

  • 第三种：根据指针的简洁版

字符串反转

  • 第一种：根据下标进行反转

  • 第二种：根据指针进行反转

字符串格式化

  • int sprintf(char * str , const char * format,…), 失败返回-1，成功返回格式化字符个数

  • 参数1：格式化的后的目标字符串 参数2：格式化形式 参数3：… 格式化中的参数

  • 字符串拼接
